# 需求文档：右键上下文菜单 (Context Menu)

> Phase 1 - 需求 3 | 优先级: P0

## 1. 背景

当前画布右键被 `oncontextmenu = preventDefault()` 拦截，无任何交互。用户需要通过工具栏或键盘快捷键执行常用操作。右键菜单是设计工具的基础交互，能大幅提升操作效率。

## 2. 目标

右键点击画布或元素时弹出上下文菜单，提供常用操作的快捷入口。

## 3. 功能需求

### 3.1 菜单触发

- 右键点击画布空白区域 → 显示画布菜单
- 右键点击选中元素 → 显示元素菜单
- 右键点击未选中元素 → 先选中该元素，再显示元素菜单
- 右键点击多选中的某个元素 → 显示多选菜单
- 点击菜单外部或按 Escape → 关闭菜单

### 3.2 画布菜单（空白区域右键）

| 菜单项 | 快捷键 | 操作 |
| --- | --- | --- |
| 粘贴 | Ctrl+V | 粘贴剪贴板内容 |
| 全选 | Ctrl+A | 选择所有元素 |
| — | — | 分割线 |
| 缩放到适合 | — | 缩放视口适应全部内容 |
| 缩放到 100% | — | 重置缩放到 100% |

### 3.3 单选元素菜单

| 菜单项 | 快捷键 | 操作 |
| --- | --- | --- |
| 复制 | Ctrl+C | 复制选中元素 |
| 粘贴 | Ctrl+V | 粘贴 |
| 原地复制 | Ctrl+D | 原地复制元素 |
| 删除 | Delete | 删除选中元素 |
| — | — | 分割线 |
| 置于顶层 | — | 移到最上层 |
| 置于底层 | — | 移到最下层 |

### 3.4 多选元素菜单

| 菜单项 | 快捷键 | 操作 |
| --- | --- | --- |
| 复制 | Ctrl+C | 复制 |
| 粘贴 | Ctrl+V | 粘贴 |
| 原地复制 | Ctrl+D | 原地复制 |
| 删除 | Delete | 删除 |
| — | — | 分割线 |
| 置于顶层 | — | 置顶 |
| 置于底层 | — | 置底 |

### 3.5 菜单样式

- 白色背景，圆角阴影，与 Figma 风格一致
- 菜单项高亮悬停效果
- 快捷键提示右对齐灰色显示
- 分割线用于逻辑分组
- 菜单出现在鼠标右键点击位置
- 菜单不超出画布边界（自动调整方向）

### 3.6 交互细节

- 点击菜单项后菜单立即关闭
- 菜单打开期间不响应画布其他交互（选择、拖拽等）
- 菜单使用屏幕坐标定位（不受画布缩放平移影响）

## 4. 验收标准

- [ ] 空白区域右键显示画布菜单
- [ ] 选中元素右键显示元素菜单
- [ ] 未选中元素右键，先选中再显示菜单
- [ ] 多选右键显示多选菜单
- [ ] 菜单项点击执行对应操作
- [ ] 快捷键提示正确显示
- [ ] 点击外部 / Escape 关闭菜单
- [ ] 菜单不超出画布边界
