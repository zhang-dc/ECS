# 需求文档：旋转交互 (Rotation Interaction)

> Phase 1 - 需求 1 | 优先级: P0

## 1. 背景

当前 LayoutComponent 已有 `rotation` 字段（弧度），LayoutSystem 会将其同步到渲染层（Pixi DisplayObject.rotation），RightPanel 也有旋转角度输入框。但缺少**鼠标交互旋转**——用户无法通过拖拽手柄来旋转元素，只能手动输入角度值。

Figma 的旋转交互：选中元素后，鼠标悬停在四角手柄**外侧**区域时出现旋转光标，拖拽即可旋转元素。

## 2. 目标

实现选中元素后，通过鼠标拖拽四角外侧区域旋转元素的交互。

## 3. 功能需求

### 3.1 旋转区域检测

- 四个角手柄（tl/tr/br/bl）外侧区域为旋转热区
- 热区范围：距角手柄中心 `HANDLE_HIT_TOLERANCE ~ HANDLE_HIT_TOLERANCE * 2` 的环形区域
- 即：比 resize 手柄命中区域更远，但仍在合理范围内

### 3.2 旋转行为

- **旋转中心**: 元素（或多选包围盒）的几何中心
- **旋转角度**: 指针相对旋转中心的角度变化量（当前角度 - 按下时角度）
- **Shift 约束**: 按住 Shift 时，旋转角度吸附到 15° 的整数倍
- **实时更新**: 拖拽过程中实时更新 `LayoutComponent.rotation` 和渲染

### 3.3 光标反馈

- 悬停旋转区域时：显示旋转光标（自定义 CSS `grab` 或使用 `url()` 自定义图标，先用 `crosshair` 做 MVP）
- 旋转拖拽中：保持旋转光标

### 3.4 角度显示

- 旋转拖拽过程中，RightPanel 的旋转输入框实时更新（通过 ENTITY_MOVE 事件）

### 3.5 单选 vs 多选

- **单选**: 旋转单个元素，中心为元素 AABB 中心
- **多选**: 暂不支持多选旋转（复杂度高，需要绕组中心旋转每个元素的位置+角度）。多选时只显示 resize 手柄，不显示旋转区域

### 3.6 Undo 支持

- 旋转结束时触发 `ENTITY_MOVE` 事件，已有的 History 系统会记录状态变化

## 4. 非功能需求

- 旋转检测不影响 resize 手柄的命中（resize 优先级高于旋转）
- 旋转过程中不触发选择/拖拽

## 5. 验收标准

- [ ] 单选元素，鼠标移到角手柄外侧区域，光标变化
- [ ] 从旋转区域开始拖拽，元素绕中心旋转
- [ ] 按住 Shift，角度吸附 15° 步进
- [ ] 旋转过程中属性面板角度值实时更新
- [ ] 旋转结束后可以 Undo 撤回
- [ ] 多选时不触发旋转
- [ ] 旋转后 resize 手柄仍能正常工作
- [ ] 旋转后选中框跟随元素旋转
