# 需求文档：空格+拖拽平移 (Space + Drag Pan)

> Phase 1 - 需求 5 | 优先级: P0

## 1. 背景

当前画布平移视口有两种方式：鼠标滚轮和切换到 Hand 工具 (H)。但 Figma/Sketch 等编辑器的核心交互习惯是**按住空格键临时激活手型工具**，释放后自动恢复原工具，无需手动切换。这是设计师高频使用的操作，缺失会严重影响使用效率。

## 2. 目标

按住空格键时临时切换为手型模式，可拖拽平移画布；释放空格后自动恢复到之前的工具模式。

## 3. 功能需求

### 3.1 核心交互流程

1. 用户在任意工具模式下按住空格键
2. 画布进入临时手型模式，光标变为 `grab`
3. 按住空格 + 鼠标左键拖拽 → 平移视口，光标变为 `grabbing`
4. 释放空格键 → 恢复之前的工具模式和光标
5. 如果先释放鼠标再释放空格，平移停止，光标回到 `grab`，释放空格后恢复原工具

### 3.2 约束条件

- **文本编辑中不生效**：当 `world.isTextEditing === true` 时，空格键应正常输入空格字符，不触发手型模式
- **绘制过程中不生效**：当 `toolComponent.isDrawing === true` 时（正在绘制 rect/circle），空格不触发
- **已经是 Hand 工具时无需处理**：用户主动切换到 Hand 工具后，空格键不产生额外效果
- **不改变工具面板 UI 状态**：临时手型模式不触发 `TOOL_CHANGE` 事件，UI 上工具面板仍显示原工具高亮

### 3.3 光标行为

| 状态 | 光标 |
| --- | --- |
| 空格按住（未拖拽） | `grab` |
| 空格按住 + 拖拽中 | `grabbing` |
| 释放空格 | 恢复原工具对应光标 |

### 3.4 边界情况

- 快速按放空格（tap）不应产生可感知的闪烁
- 空格按住期间切换浏览器标签再回来，应正确恢复状态
- 空格按住期间如果有其他键盘快捷键（如 Ctrl+Z），不应冲突

## 4. 验收标准

- [ ] Select 工具下按住空格，光标变为 grab
- [ ] 空格 + 拖拽，画布平移，光标变为 grabbing
- [ ] 释放空格，恢复 Select 工具的 default 光标
- [ ] Rect 工具下按住空格（非绘制中），可临时平移
- [ ] 文本编辑中按空格，正常输入空格字符，不触发平移
- [ ] 绘制 rect/circle 过程中按空格，不触发平移
- [ ] 工具面板 UI 不跟随临时手型模式变化
- [ ] 已是 Hand 工具时，空格无额外效果
